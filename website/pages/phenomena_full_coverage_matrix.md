## Purpose

This page is the "all-phenomena" execution map for COG.
It is intentionally explicit about:
1. what we think the mechanism is,
2. what is actually evidenced today,
3. what blocks closure,
4. what single next test would move each lane forward.

Status legend:
1. `supported_bridge`: strong structural closure with explicit bridge assumptions still open.
2. `partial`: meaningful evidence exists, but derivation is incomplete.
3. `active_hypothesis`: mechanism proposed, validation lane active.
4. `open`: no claim-grade mechanism yet.

## Axiom Scope

Current model profiles:
1. `integer` kernel profile (canonical for promotion).
2. `unity` and `projective_unity_v1` profiles (exploratory unless separately promoted).

Hard policy:
1. Do not mix kernel profiles inside one claim closure.
2. Every promotion lane must freeze update-rule profile, observable contract, and acceptance gates before execution.

## Cross-Cutting Risks (Most Likely to Break Closure)

1. Profile mixing: using integer and unity/projective outputs in one closure record.
2. Policy shopping: changing running/plateau/normalization rules after seeing results.
3. Scope inflation: promoting structural matches as full numeric derivations.
4. Initialization leakage: relying on one handpicked start state without ensemble robustness.
5. Observable drift: changing metric definitions between preregistration and reporting.

## What "Explain Every Phenomenon" Means Operationally

For this lab, a phenomenon is only "explained" when:
1. mechanism is stated in one locked contract,
2. observable is predeclared,
3. deterministic artifacts are replayable,
4. falsification conditions are explicit and tested,
5. skeptic review confirms scope and limits,
6. claim status reflects evidence quality without overreach.
